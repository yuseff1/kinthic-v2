# Generate Tests Workflow

## When to use
Run this workflow when asked to write unit tests or integration tests for existing code.

## Workflow
1. Identify the public API of the target module.
2. Determine the core success paths.
3. Determine edge cases, empty states, and failure paths.
4. Write test assertions using standard frameworks (pytest).
5. Mock external dependencies.

## Output
Executable test code matching the repository's testing conventions.
