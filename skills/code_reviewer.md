# Code Review Workflow

## When to use
Run this workflow when asked to review a PR, commit, or diff.

## Workflow
1. Analyze the changes for correctness and logic errors.
2. Check for adherence to style guides and best practices.
3. Identify performance bottlenecks.
4. Verify tests are updated or added.

## Output
Provide inline review comments and an overall summary of the code quality.
