# Issue Triage Workflow

## When to use
Run this workflow when asked to triage GitHub issues or bug reports.

## Workflow
1. Read the issue description to understand the bug/feature.
2. Attempt to reproduce the issue mentally or locally.
3. Assign labels (bug, enhancement, duplicate, won't fix).
4. Identify the subsystem/module responsible.

## Output
Reply to the issue with clarifying questions or a preliminary diagnosis.
