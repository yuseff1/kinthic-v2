# Refactor Code Workflow

## When to use
Run this workflow when asked to refactor, simplify, or clean up code.

## Workflow
1. Identify the code smell (e.g., long methods, duplicate code, tight coupling).
2. Propose a structural change (extract method, introduce class, use design pattern).
3. Ensure the public interface remains unchanged.
4. Execute the refactoring safely.

## Output
The refactored code and a brief explanation of the improvements.
