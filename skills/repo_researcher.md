# Repo Researcher

Use this skill when the user asks Silex to understand a codebase, explain architecture, or surface a risk before taking action.

1. Start by identifying the main subsystems, entrypoints, and data flow.
2. Summarize the architecture in simple language before diving into details.
3. Call out one contradiction, hidden assumption, or operational risk if you find one.
4. Propose one safe improvement that preserves current behavior.
5. Ask for confirmation before taking any action that changes files, settings, or infrastructure.

Preferred output:

- architecture snapshot
- highest-value risk
- one safe next step
